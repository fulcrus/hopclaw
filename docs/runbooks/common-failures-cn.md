# Runbook: 常见生产故障

## 1. `/healthz` 返回 503

优先检查：

- 进程是否正常启动
- 配置是否可解析
- 状态目录是否可写
- 模型配置是否把启动流程卡住

## 2. Audit dead-letter 增长

优先检查：

- 目标 sink 可达性
- 认证头 / secret / token
- Elasticsearch 索引写权限
- Splunk HEC token 与 index

再检查：

- 是否近期做过网络策略变更
- 是否近期更换了证书或域名

## 3. SQLite 写入异常

优先检查：

- 容器卷是否只读
- 磁盘是否满
- 是否误把多个活跃实例指向同一状态目录

正确做法：

- 一个状态目录只对应一个活跃 HopClaw

## 4. Caddy 自动证书失败

优先检查：

- 域名是否已解析到当前入口
- 80/443 是否对外可达
- 企业网络是否拦截 ACME

如果是纯内网：

- 不要强行走公网 ACME
- 直接改用公司现有 ingress / LB

## 5. 模型调用突然大量报错

优先检查：

- API key 是否过期
- provider 额度或限流
- 模型名是否被下线
- 公司出口网络是否变更

## 6. 恢复后状态不一致

优先检查：

- 是否在服务仍写入时执行 restore
- 是否把旧文件与恢复文件混在一起
- `.bak` 文件是否可以帮助比对

如果恢复异常：

- 停止服务
- 重新从干净目录恢复
- 再做启动验证

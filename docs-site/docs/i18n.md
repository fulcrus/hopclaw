# Internationalization

HopClaw now ships product-facing locale support for:

- `en`
- `zh-CN`
- `ja-JP`

## What is localized

- backend message catalog entries
- dashboard UI dictionaries
- onboarding and setup copy
- browser locale metadata

This documentation site itself is English-canonical. Translations of long-form
docs are not maintained in lockstep with the English content; for accuracy,
read the English version when in doubt.

## Set locale from config

```yaml
locale: ja-JP
```

## Set locale from environment

HopClaw checks locale hints in this order:

1. `HOPCLAW_LOCALE`
2. `LC_ALL`
3. `LANG`

Examples:

```bash
export HOPCLAW_LOCALE=ja-JP
hopclaw serve
```

```bash
export LANG=ja_JP.UTF-8
hopclaw serve
```

## Browser-side language switching

The dashboard language toggle cycles through:

```text
en → zh-CN → ja-JP
```

The selected language is kept in local storage and the document language is updated to the correct locale tag.

## Translation strategy

HopClaw uses:

- YAML catalogs for backend strings
- module-based JS dictionaries for the web dashboard
- locale normalization for values like `ja`, `ja-JP`, and `ja_JP`

## Recommendation

If you operate a shared instance:

- set a stable default `locale:` in config
- let operators switch languages in the dashboard for personal preference
- keep API field names and automation identifiers in canonical English
